Enhancement One - Software Design Narrative

The artifact selected for this enhancement is a Python-based Inventory Management Program originally created during earlier coursework in the Computer Science program. The program is a console-based application that allows users to add items, view inventory, sort items, and search for specific items. The original version of the program demonstrated foundational programming concepts such as functions, loops, conditionals, and basic user input handling.
This artifact was selected because it represents a strong foundation in programming and provides clear opportunities for improvement in software design and engineering. The simplicity of the original program made it an ideal candidate for enhancement, particularly in the areas of structure, maintainability, and reliability. The artifact showcases my ability to build functional applications while also demonstrating my ability to analyze existing code and improve it using better software design practices. The enhancements highlight important software engineering concepts such as input validation, error handling, and improved program organization.
Several enhancements were implemented to improve the overall software design and functionality of the program. Error handling was added to prevent the program from crashing when users enter invalid input, such as non-numeric values for item quantity. This improves the reliability and robustness of the application. Input validation was introduced to ensure that users cannot enter empty values for item names or search inputs, improving data integrity and overall usability. The program structure was improved by adding a main function and organizing the program flow more clearly, aligning the program with standard software development practices and making it easier to maintain and extend. Comments were also added throughout the code to improve readability and documentation, making the program easier to understand for other developers.
The enhancements implemented in this artifact align with Course Outcomes 3 and 4. These improvements demonstrate the ability to design and evaluate computing solutions using appropriate software development practices. The addition of error handling, validation, and improved structure reflects the use of well-founded techniques in computing. The enhanced program is more reliable, maintainable, and aligned with industry standards, demonstrating progress toward professional-level software development skills.
Through the process of enhancing this artifact, I learned the importance of writing code that is not only functional but also maintainable and robust. While the original program worked as intended, it lacked proper error handling and structure, which are critical in real-world applications. One of the main challenges was determining how to improve the design without overcomplicating the program. This required balancing simplicity with functionality while ensuring that the enhancements were meaningful and aligned with software engineering principles. Overall, this experience strengthened my understanding of software design and reinforced the importance of building applications that are scalable, reliable, and user-friendly.
