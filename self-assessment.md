## Data Structures and Algorithms

My enhancement of the inventory management system for the Algorithms and Data Structures category demonstrates my ability to reason about computational efficiency and make deliberate design choices. I replaced a linear search implementation with a binary search algorithm, reducing the time complexity of item lookups from O(n) to O(log n), and implemented an insertion strategy that maintains a persistently sorted list to ensure binary search operates correctly at all times. These decisions reflect an understanding of trade-offs: the minor overhead of maintaining sort order on insertion is justified by the significant efficiency gains at scale. This kind of algorithmic thinking is directly applicable to financial applications where performance and data integrity are non-negotiable.

## Software Engineering and Databases

My software engineering enhancement focused on restructuring the original program using sound design principles: modular function separation, a clearly defined entry point, input validation, and meaningful error handling. These improvements make the codebase more robust, readable, and maintainable. My database enhancement introduced persistent storage using SQLite, implementing full CRUD operations and parameterized queries to prevent SQL injection. Together, these enhancements demonstrate the ability to take a functional but fragile piece of software and evolve it into something that meets professional standards for structure, reliability, and data integrity.

---

## Security Mindset

Security was a deliberate consideration throughout my enhancements. In the database artifact, I implemented parameterized queries rather than string interpolation — a direct mitigation against SQL injection attacks, which remain among the most common and damaging vulnerabilities in data-driven applications. I also incorporated input validation at every point of user interaction to ensure that malformed or malicious input cannot produce unintended system behavior. Developing a security mindset means anticipating how a system can be exploited before it is, and that principle guided my design decisions throughout this capstone.

## Summary of Artifacts

The three artifacts in this portfolio are enhancements of a single original Python inventory management program, each demonstrating growth in a distinct area of computer science. The first enhancement addresses software design and engineering, refactoring the original code for modularity, readability, and robustness. The second enhancement addresses algorithms and data structures, replacing inefficient search logic with a binary search implementation operating on a maintained sorted list. The third enhancement addresses databases, replacing volatile in-memory storage with a persistent SQLite database secured against injection vulnerabilities. Taken together, these artifacts demonstrate my ability to iteratively improve a software system across multiple dimensions — engineering quality, algorithmic efficiency, and data persistence — which is precisely the kind of full-spectrum thinking that distinguishes a computer science professional in any industry.
